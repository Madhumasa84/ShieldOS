Add the real-time DNS processing pipeline:

1. DNS QUERY ENDPOINT (high performance):
POST /api/v1/dns/query
Headers: Authorization: Bearer {device_token}
Body: { "domain": "ads.doubleclick.net", "device_id": "xxx" }
Response: {
  "blocked": true,
  "category": "ads",
  "rule": "StevenBlack",
  "response_time_ms": 2
}
- Must respond in under 20ms
- In-memory LRU cache (50,000 entries)
- Async log to threat_events table
- Never block the response waiting for DB write

2. SUBDOMAIN MATCHING:
- "ads.google.com" should match if "google.com" blocked
- "tracking.evil.ads.com" matches "ads.com"
- Wildcard support: *.doubleclick.net blocks all subdomains
- Exact match checked first (faster)
- Parent domain fallback second

3. BATCH DNS CHECK:
POST /api/v1/dns/batch
Body: { "domains": ["domain1.com", "domain2.com"], "device_id": "xxx" }
Response: { "results": { "domain1.com": true, "domain2.com": false } }
Max 100 domains per batch request
